## RecordingCore — 記錄核心

- **功能：**
    - 支出、收入、轉帳三種交易類型的建立、編輯、軟刪除
    - 定期交易範本管理與自動補產生
    - 帳戶合併與類別合併
    - 全域操作撤銷，支援短時間倒數還原
- **目的：**
    - 使用者寫入帳本的操作集合——包含所有建立、修改、刪除、補正財務紀錄的行為
- **做法：**
    - 所有寫入即時落地本地資料庫
    - Undo 採立即寫入加反向操作策略
    - 定期交易補產生於 App 啟動時執行，優先於批次同步
- **排除：**
    - 銀行 API 直接讀取交易資料
    - AI 自動分類建議，屬 Cloud Service / AIAdvisor 範疇
- **利弊：**
    - 本地寫入確保離線環境可正常記帳，但跨裝置一致需升級付費同步
    - Undo 提供操作安全感，但短時間窗口關閉後無法還原，需配合批次操作謹慎使用

### TransactionEditor — 交易編輯器

- **功能：**
    - 建立、編輯、軟刪除支出與收入兩種交易類型
    - 刪除後短時間內可 Undo 還原
- **目的：**
    - 作為使用者輸入財務資料的主要操作入口
- **做法：**
    - 停用與刪除功能統一在 Editor 管理，不使用 List 畫面的左滑操作
    - 編輯定期交易時，透過對話框讓使用者選擇變更範圍為僅此一筆或此筆及未來
    - Undo 採用立即寫入加反向操作策略
- **排除：**
    - List 畫面的左滑刪除
- **利弊：**
    - 統一 Editor 操作避免手勢衝突，且職責劃分清晰；但相比左滑操作多一個點擊步驟

### TransferEditor — 轉帳編輯器

- **功能：**
    - 建立、編輯、軟刪除帳戶間轉帳
    - 跨幣別轉帳時自動從金額推算並記錄隱含匯率
- **目的：**
    - 讓使用者記錄資金在帳戶間的流動，並自動維護跨幣別的匯率歷史
- **做法：**
    - 轉帳模型儲存 amountFrom、amountTo 與 impliedRate
    - 跨幣別轉帳完成後自動建立對應的 CurrencyRate 記錄
    - 編輯定期轉帳時，透過對話框讓使用者選擇變更範圍為僅此一筆或此筆及未來
- **排除：**
    - 多步跳轉換算——Strict Direct Rate 策略，不支援中間幣別
- **利弊：**
    - 自動記錄隱含匯率降低維護成本，但僅支援直接匯率，若無外幣對基礎貨幣直接匯率則無法換算報表

### RecurringTransactions — 定期交易

- **功能：**
    - 定期交易範本的建立、編輯、刪除，支援僅此筆或此筆之後兩種範圍
    - App 啟動時自動補產生遺漏的定期交易實例
- **目的：**
    - 讓固定支出與收入自動記錄，減少重複手動輸入
- **做法：**
    - 定期交易補產生模組於 App 啟動時執行，依上次檢查時間對照當日時區補產生所有遺漏交易
    - 產生作業優先於批次同步執行
- **排除：**
    - 複雜條件式定期規則，例如每月最後一個工作日
- **利弊：**
    - App 啟動觸發確保離線環境也能自動產生，但長期未開 App 時會一次補產生大量紀錄
    - 付費期間建立的定期排程，訂閱過期後仍持續產生實例，屬已購買功能的延續而非漏洞

### MergeOperation — 合併操作

- **功能：**
    - 帳戶合併：將來源帳戶的所有交易與轉帳遷移至目標帳戶後軟刪除來源
    - 類別合併：將來源類別的所有交易遷移至目標類別後軟刪除來源
    - 合併後短時間內可 Undo 還原
- **目的：**
    - 讓使用者整理重複或多餘的帳戶與類別，維護資料結構整潔
- **做法：**
    - 合併服務批次更新所有關聯紀錄的帳戶或類別對應關係
    - 來源與目標為同一帳戶的自我轉帳邊緣情況，自動軟刪除以避免冗餘紀錄
- **排除：**
    - 合併後自動去重複交易紀錄
- **利弊：**
    - 批次遷移保留完整歷史紀錄，但合併後若需拆回只能透過 Undo，不支援事後分割

### UndoSystem — 撤銷系統

- **功能：**
    - 全域操作撤銷，短時間倒數自動消失，支援手動提前關閉
    - 支援交易、轉帳、帳戶、類別、合併五種撤銷類型
- **目的：**
    - 提供操作安全網，降低使用者因誤操作造成的資料損失
- **做法：**
    - 全域撤销狀態集中管理；每次寫入操作完成後將反向操作登記至撤销佇列；浮動操作列顯示撤销按鈕
- **排除：**
    - 多層級歷史撤銷；已完成同步的操作無法撤銷
- **利弊：**
    - 立即撤銷的體驗直覺易用，但時間窗口關閉後無法還原，需配合合併等批次操作謹慎使用
