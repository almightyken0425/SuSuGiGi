## Auth / AppClient — 身份驗證用戶端

- **功能：**
    - Google 帳號登入與登出。
    - 首次登入時自動建立預設帳戶與類別。
- **目的：**
    - 提供使用者統一身份，作為雲端同步與付費授權的基礎。
- **做法：**
    - Firebase Authentication 管理 Google 登入狀態；首次登入後自動建立預設帳戶與類別至本地資料庫。
- **排除：**
    - Apple ID 登入，因資料衝突問題暫不實作；其他第三方社交登入。
- **利弊：**
    - Firebase Authentication 支援 Google 帳號登入，免費使用。
    - Apple ID 登入時使用者可選擇不分享 Email，造成帳號合併困難，故先排除。
    - 保留主動登出按鈕，提供切換帳號的彈性；需處理本地資料與不同帳號間的隔離問題，但明確的登出選項能提供安全感。

---

### LoginFlow — Google 登入流程

- **功能：**
    - Google Sign-In 介面。
    - Firebase Authentication 登入與登出。
- **目的：**
    - 讓使用者透過已有的 Google 帳號快速完成登入，無需額外註冊流程。
- **做法：**
    - Firebase 登入套件整合；登入後取得身份憑證，供後續雲端資料存取驗證使用。
- **排除：**
    - 電子郵件加密碼登入；Apple ID 登入。

---

### PostAuthSetup — 首次登入初始化

- **功能：**
    - 首次登入時自動建立預設帳戶與類別集。
- **目的：**
    - 降低新使用者上手門檻，讓記帳可立即開始。
- **做法：**
    - 首次登入初始化邏輯於系統偵測到新使用者後觸發，將預設資料集寫入本地資料庫。
- **排除：**
    - 使用者自訂初始資料集。
- **利弊：**
    - 預設資料集加速上手，但使用者仍需手動調整以符合個人習慣。
