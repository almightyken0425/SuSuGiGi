## AppSetting — 應用設定

- **功能：**
    - 資料管理：帳戶與類別的 CRUD、CSV 資料匯入與匯出、清空資料庫。
    - 偏好設定：外觀主題、貨幣與財務設定、語言與時區、帳號操作。
    - 方案：付費升級入口，免費版限定顯示。
- **目的：**
    - 提供使用者統一的設定入口。
- **做法：**
    - 應用設定頁面作為模組根頁面，以 Section 分組呈現三大功能區塊；各子頁面以 Stack 導航進入。
- **排除：**
    - 系統層級設定，例如：通知、隱私權等，由 iOS/Android 系統設定管理。

---

### DataManagement — 資料管理

- **功能：**
    - 帳戶與類別的進入點，導向各自的 CRUD 頁面。
    - 資料匯入匯出 CSV 格式。
    - 清空資料庫。
- **目的：**
    - 讓使用者集中管理記帳的基本資料結構，以及進行資料備份與還原。
- **做法：**
    - 資料管理頁面提供三個入口：類別管理、帳戶管理、資料操作；資料操作頁面再拆分為匯入、匯出、危險操作三個子區域。
- **排除：**
    - 帳戶餘額自動從銀行拉取；AI 自動分類建議。

---

#### CategoryCRUD — 類別管理

- **功能：**
    - 建立、編輯、停用類別。
    - 預設類別集隨首次登入自動建立。
- **目的：**
    - 讓使用者依個人消費習慣組織財務分類，支撐報表統計準確度。
- **做法：**
    - 預設資料集於初始化時自動建立；自訂類別付費 Tier 1 後可無限新增；停用不刪除，歷史交易保留關聯。
- **排除：**
    - AI 自動分類建議，屬 Cloud Service / AIAdvisor / Backend 負責範疇。

---

#### AccountCRUD — 帳戶管理

- **功能：**
    - 建立、編輯、停用帳戶。
    - 設定帳戶幣別。
- **目的：**
    - 讓使用者將資產對應到真實的銀行帳戶、現金或投資帳戶。
- **做法：**
    - 本地資料庫儲存；停用不刪除，歷史交易保留關聯。
- **排除：**
    - 帳戶餘額自動從銀行拉取。

---

#### DataOperations — 資料操作

- **功能：**
    - CSV 匯入：交易、轉帳、定期交易。
    - CSV 匯出：交易、轉帳、定期交易。
    - 清空資料庫：刪除所有本地資料。
- **目的：**
    - 讓使用者可以透過 CSV 進行資料備份、還原或從其他工具遷移資料；清空資料庫提供重置能力。
- **做法：**
    - 匯入採五步驟 Wizard: 模板說明 → 選擇檔案 → 欄位對應 → 內容比對 → 預覽確認
    - 匯出生成 CSV 並呼叫系統分享介面
    - 清空資料庫需批次刪除所有本地資料，操作前二次確認。
- **排除：**
    - 雲端自動備份由 CloudSync 負責
    - 匯入格式目前僅支援 CSV，不支援 OFX、QIF 等格式。

### Preference — 偏好設定

- **功能：**
    - 外觀：配色主題、啟動模式。
    - 貨幣與財務：基礎貨幣、幣別顯示格式、匯率管理。
    - 語言與時區：語言切換、時區設定。
    - 帳號：登入或登出操作。
- **目的：**
    - 管理所有影響全 App 呈現與行為的個人化偏好設定。
- **做法：**
    - 偏好狀態集中管理，持久化於本地資料庫
    - 設定變更即時反映，並由同步引擎同步至雲端供跨裝置一致
    - 多裝置間以 Last-Write-Wins 策略決定最終狀態。
- **排除：**
    - 使用者自訂調色盤
    - 自動依裝置地區選擇時區

#### Appearance — 外觀設定

- **功能：**
    - ThemeSystem：主題切換介面。
    - LaunchModeSetting：設定 App 啟動後預設導航目標
      - 首頁
      - 支出
      - 收入
      - 轉帳
- **目的：**
    - 透過視覺設計降低記帳焦慮，增加日常使用黏著度
    - 讓高頻記帳的使用者省去每次點擊的步驟
- **做法：**
    - 主題設計以全域 Token 管理
    - 確保跨頁面顏色、字體、間距一致，禁止寫死
    - 啟動模式偏好設定持久化於本地，App 啟動後依設定決定初始導航
- **排除：**
    - 使用者自訂調色盤
    - 依時間或情境自動切換啟動模式

#### CurrencyAndFinance — 貨幣與財務設定

- **功能：**
    - 基礎貨幣設定：設定報表計算使用的基礎貨幣。
    - 幣別顯示格式設定：設定各幣別的小數位數與千位單位顯示，逐幣別配置；點入單一幣別可細部編輯。
    - 匯率管理：手動輸入並儲存外幣對基礎貨幣的匯率，可新增或編輯單筆匯率記錄。
- **目的：**
    - 讓多幣別帳戶的報表可以統一換算為使用者熟悉的貨幣呈現
    - 確保各幣別依慣例正確呈現金額格式
    - 維持完整離線匯率管理能力
- **做法：**
    - 基礎貨幣偏好持久化於本地設定
    - 各幣別顯示格式逐幣別獨立配置，全 App 統一取用
    - 匯率採 Strict Direct Rate——僅使用外幣對基礎貨幣的直接匯率計算報表，不支援中間跳轉
    - 匯率採追加記錄策略，可透過新增記錄修正歷史
- **排除：**
    - 外部即時匯率 API
    - 同時支援多種基礎貨幣檢視
    - 自動依 ISO 4217 標準套用格式
- **利弊：**
    - 完全離線可用，且匯率可反映使用者實際換匯成本，但使用者需自行維護，若疏於更新則報表數字失真。
    - Strict Direct Rate 不支援跳轉，若無外幣到基礎貨幣的直接匯率，該筆資產在報表中無法計算。
    - 單一基礎貨幣使報表計算邏輯簡單，但使用者若同時操作多種主要貨幣，只能選擇其中一種作為基準。

#### RegionAndLanguage — 語言與時區設定

- **功能：**
    - LanguageSetting：應用程式語言切換——繁體中文與英文。
    - TimeZoneSetting：時區設定，影響日期邊界計算。
- **目的：**
    - 確保報表日期邊界與使用者所在地區一致，並以母語呈現介面。
- **做法：**
    - 語言切換透過本地化框架管理；時區以標準時區識別碼（IANA 格式）儲存，日期邊界計算依照該設定執行。
- **排除：**
    - 自動依裝置地區選擇語言或時區。
- **利弊：**
    - 明確時區設定避免跨時區記帳時日期錯誤，但使用者需主動設定，預設值可能不符當前所在地。

---

#### Account — 帳號操作

- **功能：**
    - 登出系統。
- **目的：**
    - 讓使用者可以登出當前 Google 帳號。
- **做法：**
    - 顯示單一登出按鈕；登出前以二次確認對話框防止誤操作，登出後由 Firebase Auth 清除憑證並觸發登入牆。
- **排除：**
    - 帳號刪除操作。
- **利弊：**
    - 明確的登出選項能保護隱私避免他人存取。

### Plan — 方案升級入口

- **功能：**
    - 免費版用戶限定顯示的付費升級入口，導向付費牆頁面。
    - 付費後此 Section 自動隱藏。
- **目的：**
    - 在設定頁提供低門檻的升級入口，讓免費用戶在操作設定時也能觸及升級路徑。
