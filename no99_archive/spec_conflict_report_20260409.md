## 🟡 流程與細節定義出入

### 首次登入初始化流程分歧

- **首次登入流程規範:**
  - 範例程式碼將剛註冊的使用者資料直接寫入本地 `users` collection
- **登入後邏輯規範:**
  - 新使用者轉正需覆蓋寫入本地 `Settings` 表，並創建現金帳戶與預設交易類別
  - 僅在 Premium 為 True 時才將資料寫入 Firestore `users/{uid}`
- **影響:**
  - 新手上船的本地資料初始化步驟分散於兩份規範，且作法分歧

### 語言代碼規格不一致

- **偏好設定更新規範:**
  - 繁體中文語言代碼為 `zh-Hant`
- **使用者資料結構定義:**
  - 繁體中文介面語言填寫為 `zh-TW`
- **影響:**
  - 切換語言時，代碼無法對應到正確的語言設定

### 配額管理執行粒度描述落差

- **雲端同步模組說明:**
  - 超額行為描述為超出每日上限時暫停同步，未區分讀寫方向
- **配額邏輯規範:**
  - 寫入超標時跳過 Push、保留 Pull
  - 讀取超標時跳過 Pull、保留 Push
- **影響:**
  - 規劃文件過於概括，開發與 QA 可能誤以為任何超額都導致同步全面停擺

### 基礎貨幣欄位格式不統一

- **使用者資料結構定義 — Firestore:**
  - 欄位名稱 `currency`，型別 String，儲存 ISO 4217 代碼如 TWD
- **資料模型定義 — 本地 DB:**
  - 欄位名稱 `baseCurrencyId`，型別 Number，儲存 ISO Numeric ID
- **影響:**
  - 轉換邏輯分散於各規範，缺乏統一的格式映射定義

### Undo 範圍判斷標準缺失

- **撤銷操作規範:**
  - 已完成同步的操作無法撤銷
- **衝突點:**
  - 已完成同步的判斷標準未定義
  - 多裝置 LWW 衝突解決後，失敗方是否仍可撤銷未說明
- **影響:**
  - 多裝置場景下的撤銷行為不一致

### Settings 更新時間戳誤觸發增量同步

- **登入後邏輯規範:**
  - 登入時覆蓋寫入 Settings 並將 `updatedOn` 設為當下時間
- **批次同步規範:**
  - 增量同步查詢條件為 `updatedOn > lastSyncTimestamp`
- **影響:**
  - 未實際修改的 Settings 資料在下次增量同步時被誤判為需要上傳

### 基礎貨幣變更時匯率補齊策略並存

- **偏好設定畫面規範:**
  - 基礎貨幣變更時，主動掃描外幣帳戶並建立缺失的匯率紀錄，Rate 預設為 1
- **匯率轉換邏輯規範:**
  - 查無匯率紀錄時，被動降級處理並預設為 1
- **衝突點:**
  - 兩套策略並存，無明確優先順序
- **影響:**
  - 主動建立與被動補建的執行邊界不清楚

### 匯率正向反向定義不一致

- **轉帳邏輯規範:**
  - 轉帳操作僅記錄正向匯率
- **匯率轉換邏輯規範:**
  - 查找時合併正向與反向，以時間倒序取最新紀錄
- **影響:**
  - 轉帳時的隱含匯率方向未統一，以 USD 轉 TWD 為例，記錄方向不明確

## ⚠️ 潛在風險

### Bootstrap Flow 非同步時序風險

- **應用程式啟動流程規範:**
  - 付費者背景任務中，定期交易補產生與批次同步自動觸發均列為非同步執行
  - 兩者之間無強制的執行順序依賴關係
  - 定期交易規範要求補產生優先於批次同步，但並未以強依賴方式實現
- **風險:**
  - 批次同步若先於定期交易補產生完成，當日應生成的定期交易紀錄趕不上此次同步上傳
  - 雲端資料短暫不完整，需等待下次同步條件符合才能回補

## ✅ 已解決的衝突

- **Preferences 登入覆蓋與日常 LWW 策略矛盾:**
  - 已將登入後邏輯改為 Last Write Wins 策略，並將 Preferences 整合進 SyncEngine 批次同步流程，徹底解決了離線狀態覆蓋問題
- **App 啟動時定期交易觸發權限:**
  - 定期交易檢查已從付費者背景任務移至通用核心背景任務，無須檢查 Premium 狀態
- **偏好設定同步架構:**
  - 偏好設定更新規範已採用 Write-Through 模式，優先寫本地 WatermelonDB 後直接寫入 Firestore，不經 SyncEngine
- **主題選項定義不一致:**
  - 佈景選項已抽象化為動態標識符，避免資料庫結構與前端佈景管理機制產生強耦合
- **Premium 狀態查詢呼叫時機未定義:**
  - 已於 `no1_app_bootstrap_flow.md` 新增背景同步機制，確保每次應用程式啟動過程背景呼叫 `PremiumLogic.refreshPremiumStatus()` 更新跨裝置憑證。
- **帳號切換後定期交易補產生邏輯缺失:**
  - 已於 `no12_logout_logic.md` 加入登出時強制重置 `lastRecurringCheckDate` 與 `lastSyncCheckDate` 的行為，確保帳號切換後能正常觸發這兩項檢查條件。
